# 本地视觉管线使用说明

## 运行顺序

1. 准备资产：

```bash
python -m src.prepare_assets
```

2. 运行演示：

```bash
python -m src.run_demo --video data/input/demo.mp4
```

3. 运行 CPU/GPU 几何层基准测试：

```bash
python -m src.benchmark --video data/input/demo.mp4
```

4. 单独查看本地规则决策：

```bash
python -m src.decide --snapshot outputs/keyframes/frame_0000.jpg
```

## 输出文件

- `outputs/demo_side_by_side.mp4`：原视频、深度图、语义叠加、点云俯视图的 2x2 演示视频。
- `outputs/performance.csv`：完整管线逐帧耗时。
- `benchmarks/results.csv`：CPU 反投影与 CUDA 反投影耗时和加速比。
- `outputs/keyframes/`：关键帧截图、深度图、语义图、点云预览图。
- `outputs/pointclouds/`：Open3D 写出的 PLY 点云。
- `outputs/decision/decision_report.txt`：无 API 版本的本地规则决策文本。

## 重要说明

- 当前深度为 Depth-Anything-V2 相对深度映射出的 Pseudo-LiDAR 伪距离，不是标定后的米制距离。
- 如果没有成功下载 Depth-Anything-V2 权重，程序会降级到 fallback 深度，用于验证 CUDA、语义、可视化和报告链路。
- 语义分割默认使用 `third_party/sam3/sam3.pt`，需要 `ultralytics>=8.3.237` 的 SAM 3 接口。
- 你之后拿到 Qwen-VL API key 后，设置 `DASHSCOPE_API_KEY`，再扩展 `CloudVLMDecisionClient` 即可替换本地规则决策。
- 如果你有自己的演示视频，直接覆盖 `data/input/demo.mp4` 或在命令中传 `--video your_video.mp4`。
