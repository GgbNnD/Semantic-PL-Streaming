# 效果标准与自动自评

## 较好的演示效果应该是什么样

1. 原始视频画面稳定，目标有明确前后距离关系。
2. 深度热力图连续且符合直觉：近处人/车/障碍物与远处背景有明显颜色差异，地面/道路不应大面积随机闪烁。
3. 语义叠加图能框出主要目标，误检不应主导画面；检测框区域在点云里应变成对应语义颜色。
4. 点云俯视图应持续非空，能看出近处点和远处点的层次，不能只是一团随机噪声。
5. `outputs/pointclouds/*.ply` 能用 Open3D 打开，并能旋转查看三维结构。
6. `outputs/performance.csv` 中深度、语义、滤波、投影都有耗时记录，投影后端应为 `cuda`。
7. `benchmarks/results.csv` 应显示串行 CPU 到 CUDA 有明显加速，通常应大于 8x。
8. 决策报告应说明最近目标、伪距离、是否位于中心通道，以及建议动作。

## 每次运行后的自评文件

`python -m src.run_demo` 结束后会生成：

```text
outputs/self_evaluation.md
```

它会检查：

- 演示视频是否能打开。
- 点云数量是否足够。
- 深度是否使用官方模型。
- CUDA 投影是否生效。
- CUDA 滤波是否生效。
- YOLO-World 是否正常输出。
- PLY 和决策报告是否存在。
- 稳态耗时是否过高。

`python -m src.benchmark` 结束后会生成：

```text
benchmarks/self_evaluation.md
```

它会重点检查：

- `cpu_loop_ms -> gpu_ms` 的加速比。
- NumPy 参考耗时。
- benchmark CSV 是否有效。

## 根据自评改进的优先级

1. 点云太稀：降低 `projection.stride`。
2. 运行太慢：降低 `runtime.process_width` 或提高 `projection.stride`。
3. 深度质量差：确认没有 fallback；必要时提高 `model.depth_input_size`。
4. 语义漏检：降低 `model.yolo_conf` 或换更合适的视频。
5. 点云毛刺多：调大 `filtering.depth_jump_threshold` 或保持 `filtering.enabled: true`。
6. CUDA 加速比低：使用更高分辨率或更小 stride，让并行规模更大。
7. 最终演示太短：把 `runtime.max_frames` 调到 120 或更高。
