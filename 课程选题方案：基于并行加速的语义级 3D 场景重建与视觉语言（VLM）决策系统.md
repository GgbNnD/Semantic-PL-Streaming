# 课程选题方案：基于并行加速的语义级 3D 场景重建与视觉语言（VLM）决策系统

## 1. 项目综述

本项目旨在利用单目摄像头视频流，通过深度学习模型恢复场景深度信息，并利用 CUDA 并行计算架构将 2D 像素实时投射为 3D “伪激光雷达（Pseudo-LiDAR）”点云。系统最终接入视觉语言大模型（VLM），实现从“感知”到“重构”再到“决策”的智能闭环。  



### 1.1 与人工智能的强关联

- **深度感知**：利用 Depth-Anything-V2 Large `vitl` 高精度预训练权重，从普通 RGB 图像中推理每个像素的深度值。  

  

  

- **语义理解**：集成 Ultralytics SAM 3 实现文本概念驱动的像素级语义分割，赋予 3D 空间语义标签。  

  

  

- **智能决策**：利用 Qwen2.5-VL 等大模型对重建后的 3D 环境进行逻辑推理并给出驾驶/避障建议。  

  

  

### 1.2 与并行计算的强关联

- **大规模并行投射**：将数百万像素从 2D 坐标系瞬间转换至 3D 笛卡尔坐标系，这是典型的数据并行任务。  

  

  

- **高性能核函数**：手写 CUDA Kernel（基于 Numba）优化内存访问对齐与线程调度。

- **实时流处理**：通过并行计算解决 AI 推理与几何渲染之间的算力瓶颈，实现高 FPS 推理。

## 3. 详细执行方案

### 第一阶段：AI 感知层部署（第 1 周）

1. **环境配置**：克隆官方仓库并安装依赖。  

   

   

   Bash

   ```
   git clone https://github.com/DepthAnything/Depth-Anything-V2.git
   pip install -r requirements.txt
   ```

2. **权重下载**：获取 `depth_anything_v2_vitl.pth` 高精度预训练模型，并确认 `third_party/sam3/sam3.pt` 已就位。  

   

   

3. **推理接口**：编写 Python 类调用模型，输出 1080P 的浮点型深度图数组。  

   

   

### 第二阶段：并行几何层实现（第 2 周）

1. **坐标转换数学推导**：  

   

   

   $$Z = d(u, v)$$

   $$X = \frac{(u - c_x) \cdot Z}{f_x}, \quad Y = \frac{(v - c_y) \cdot Z}{f_y}$$

2. **CUDA 核函数编写**：使用 Numba 实现 GPU 上的并行投影。

   Python

   ```
   @cuda.jit
   def back_project_kernel(depth_map, intrinsic, points_3d):
       u, v = cuda.grid(2)
       if u < depth_map.shape[1] and v < depth_map.shape[0]:
           d = depth_map[v, u]
           if d > 0.5: # 过滤无效点
               points_3d[v, u, 0] = (u - cx) * d / fx
               points_3d[v, u, 1] = (v - cy) * d / fy
               points_3d[v, u, 2] = d
   ```

### 第三阶段：功能丰富与进阶扩展（第 3 周）

- **进阶 A：并行语义上色**：接入 SAM 3 像素级分割掩码，在 GPU 内根据 Label 为点云赋予不同的 RGB 值（如车辆为红，地面为绿）。  

  

  

- **进阶 B：统计学并行滤波**：实现简单的 GPU 邻域滤波器，剔除深度突变点，提升点云平滑度。

- **进阶 C：实时可视化**：集成 Open3D 渲染器，实现 3D 视角的交互式漫游。

### 第四阶段：VLM 决策与总结（第 4 周）

1. **决策接入**：将彩色点云截图与 Prompt 发送至 Qwen2.5-VL 接口。  

   

   

   - *Prompt*: "当前 3D 重建结果显示前方有红色标注的障碍物，请根据深度信息分析碰撞风险。"  

     

     

2. **性能量化**：对比开启 CUDA 前后的 FPS 提升，记录加速比（Speedup）。

3. **演示录制**：制作对比视频，左侧为原始监控，右侧为实时生成的 3D 语义空间。

## 4. 技术栈总结表

| **模块**        | **推荐技术**       | **作用**                      |
| --------------- | ------------------ | ----------------------------- |
| **深度估计 AI** | Depth-Anything-V2  | 核心感知，获取深度信息        |
| **并行框架**    | Numba / CUDA       | 将 2D 转换 3D 的速度提升 100x |
| **语义识别**    | SAM 3（Ultralytics 8.3.237+） | 赋予空间“类别”属性            |
| **3D 处理**     | Open3D / CuPy      | 点云处理与实时渲染            |
| **认知决策**    | Qwen2.5-VL / LLaVA | 基于重建场景进行逻辑推理      |

## 5. 项目可扩展空间（针对更高评分）

- **全局 SLAM 拼接**：利用位姿估计（Pose Estimation）将多帧点云拼接成一段完整的 3D 街道地图。
- **双模型流流水线**：使用 `CUDA Streams` 让深度模型推理与坐标转换在 GPU 上异步并行运行，进一步压榨硬件性能。
- **虚拟增强现实 (AR)**：在点云空间中根据 VLM 建议动态绘制虚拟避障路径。
